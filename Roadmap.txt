# Prompt: Construir um sistema tipo Google Drive/OneDrive self-hosted

Você é um desenvolvedor sênior especialista em arquitetura de software, segurança, backend, frontend e infraestrutura DevOps.

Quero que você desenvolva um sistema completo de armazenamento de arquivos self-hosted, semelhante ao Google Drive, OneDrive ou Dropbox, para ser instalado em uma VPS Linux com bastante espaço em disco.

## Objetivo

Criar uma plataforma própria de armazenamento em nuvem, onde usuários autenticados possam gerenciar seus arquivos através de uma interface web e posteriormente sincronizar arquivos diretamente pelo computador, semelhante ao OneDrive.

O sistema deve ser modular, escalável e preparado para milhares de arquivos.

## Stack desejada

* Frontend: Next.js (App Router) + TypeScript + TailwindCSS + shadcn/ui.
* Backend: Node.js + NestJS + TypeScript.
* Banco de dados: PostgreSQL.
* ORM: Prisma.
* Armazenamento inicial: sistema de arquivos local da VPS.
* Cache e filas: Redis + BullMQ.
* Autenticação: JWT com refresh token.
* Containers: Docker e Docker Compose.
* Reverse Proxy: Nginx.
* Upload resumível: protocolo TUS.
* WebSocket: para atualização em tempo real.
* Testes: Jest.
* Documentação automática: Swagger/OpenAPI.

A arquitetura deve ser organizada em monorepo.
